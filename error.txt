1. Missing Visual C++ runtime (try first)
Install Microsoft Visual C++ Redistributable for Visual Studio 2015–2022 (x64) from Microsoft, reboot if prompted, then:

python -c "import lightgbm; print(lightgbm.__version__)"
2. Python 3.14 is too new for many wheels
Your log shows pythoncore-3.14-64. LightGBM’s Windows wheels are built for specific Python ABIs; 3.14 is often unsupported or flaky before libraries catch up.

Recommended fix: install Python 3.12.x (64-bit) from python.org, create a venv for this project, reinstall deps:

py -3.12 -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
python scripts\10_train_lightgbm.py
This matches what the README targets and avoids the same class of issues you hit with PyTorch on 3.14.

3. If it still fails after VC++ + Python 3.12
Reinstall LightGBM in that environment:

pip uninstall lightgbm -y
pip install "lightgbm>=4.0.0"
Summary: Install VC++ Redistributable x64, and prefer Python 3.12 for training on Windows; 3.14 is the likely long-term compatibility issue here.
